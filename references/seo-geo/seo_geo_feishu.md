## 一、核心观点：GEO vs SEO 三论

### 观点一：GEO 包含 SEO

**核心结论**：先把 SEO 做扎实，GEO 流量是白送的。

- ChatGPT 等 AI 会自动联网搜索
- SEO 做好的内容，GEO 流量自然来
- ChatGPT 联网搜到的内容，本质上还是那些在传统搜索引擎里表现好的内容
- **GEO 是 SEO 的子集**

### 观点二：GEO = SEO + 权威品牌建设 + "信源"站内容运营

**核心转变**：从"求着别人给我外链"变成"让 AI 主动想引用我的内容"。

- 权威品牌不只是官网权重、靠外链，而是真正的 branding marketing
- 内容要有结构化数据（Schema）
- 要有独家数据/观点
- 要够深够权威

### 观点三：GEO 很难评估 ROI

**核心建议**：GEO 重要现阶段不推荐 all in。

- GEO 没有清晰的 ROI
- 看 niche 有没有强势品牌：
  - 有强势品牌 → 等大后期再玩
  - 没有强势品牌 → 窗口期，先占坑

**关键判断**：你的 niche 有没有强势品牌

**小结**：
- 不管 SEO 还是 GEO，对系统来说都是一个挑选的过程
- 有可选的就是排名，没有就谁先出来算谁的
- 先别想太多，先把 SEO 这个打穿

## 二、GEO基础与定义

### 2.1 什么是GEO？

**GEO（Generative Engine Optimization）** 旨在创建和优化网站内容，以便生成式AI模型、AI答案引擎和AI聊天机器人能够有效地理解、抓取、并将其信息呈现给用户。

**目标平台**：
- ChatGPT、Gemini、DeepSeek、千问、豆包、元宝等
- 谷歌AI Overviews、百度AI摘要
- Perplexity、秘塔搜索等答案引擎

**核心目标**：
1. **获得有利呈现**：在AI回答中准确、正面地引用或提及品牌
2. **影响AI的输出**：让AI将您的内容视为构建答案的权威来源
3. **建立品牌权威**：通过持续被AI引用，建立行业专家形象
4. **保持高可见性**：在"零点击"环境中维持品牌曝光度
5. **保护品牌声誉**：主动管理和纠正AI对品牌的描述

### 2.2 GEO与传统SEO的根本区别

| 维度 | 传统SEO | 生成式引擎优化GEO |
|------|---------|-------------------|
| **本质** | 为算法排链接 | 为模型喂事实 |
| **目标引擎** | 传统搜索引擎算法 | 大型语言模型（LLM） |
| **核心目标** | 高排名和网站流量 | 在AI回答中的有利呈现和引用 |
| **直接受众** | 人类用户 + 搜索引擎爬虫 | 生成式AI模型（机器优先） |
| **内容焦点** | 关键词和反向链接 | 清晰度、事实准确性和结构化数据 |
| **衡量标准** | 点击率和转化率 | AI引用率和品牌提及率 |

**提炼**：
- 目标引擎不同：SEO针对传统搜索引擎算法；GEO针对大型语言模型（LLM）
- 核心目标不同：SEO追求高排名和网站流量；GEO追求在AI回答中的有利呈现和引用
- 内容焦点不同：SEO侧重关键词和反向链接；GEO侧重清晰度、事实准确性和结构化数据
- 衡量标准不同：SEO衡量点击率和转化率；GEO衡量AI引用率和品牌提及率

### 2.3 为什么说GEO是SEO的自然演进？

**共享基础**：GEO依赖于SEO的技术基础和内容质量原则

**响应变化**：GEO是为适应用户寻求直接答案的新行为而产生的演进

**扩展范畴**：GEO将优化的范围从单一网站扩展到整个网络生态中的品牌实体

**关系理解**：GEO与SEO是"SEO+GEO"的互补关系，而非"SEO vs GEO"的替代关系

### 2.4 GEO的核心目标

- **呈现优于排名**：追求在AI答案中的正面呈现，而非SERP排名
- **影响优于引流**：目标是影响AI生成的内容，而非直接为网站引流
- **权威即货币**：将品牌打造为AI眼中的行业权威
- **主动防御**：通过提供准确信息，主动管理品牌在AI中的形象

## 三、AI答案生成机制

### 3.1 RAG架构解析

**RAG（Retrieval-Augmented Generation）** 将大型语言模型的强大生成能力与实时信息检索相结合。

**工作流程**：

```
┌─────────────────────────────────────────────────────┐
│ 1. 理解意图                                           │
│    LLM分析用户查询，理解背后的真正意图                 │
├─────────────────────────────────────────────────────┤
│ 2. 信息检索（Retrieval）                              │
│    • 向量搜索：匹配意图而非字面词汇                    │
│    • 漏斗机制：召回数百候选→重排序→筛选5-10个片段     │
│    • 优化方向：围绕意图组织内容，提升事实密度和结构化   │
├─────────────────────────────────────────────────────┤
│ 3. 信息聚合与验证（Augmentation）                     │
│    • 一致性评分：多源验证，矛盾信息被惩罚              │
│    • 知识图谱：验证实体和关系                          │
│    • 优化方向：全网品牌信息保持一致，定期审计纠错        │
├─────────────────────────────────────────────────────┤
│ 4. 答案生成（Generation）                             │
│    • 增强提示词：问题 + 检索片段 + 系统指令           │
│    • 答案块：60%含无序列表，12%含有序列表             │
│    • 优化方向：结构化内容（标题、列表、表格、FAQ）     │
├─────────────────────────────────────────────────────┤
│ 5. 最终呈现                                           │
│    • 多模态：文本、图片、视频、交互元素                │
│    • 任务导向：从信息检索到任务完成                    │
│    • 优化方向：图片alt文本、视频字幕、Schema标记      │
└─────────────────────────────────────────────────────┘
```

### 3.2 AI如何选择和信任来源？

**主要评估维度**：

**主题权威性**：
- AI评估整个网站在该主题上的专业深度
- 拥有大量关于某个主题的、相互链接的深度内容（主题集群）

**EEAT信号**：
- **经验（Experience）**：第一手实践经验、真实案例研究
- **专业知识（Expertise）**：公认的专家撰写、详细作者简介
- **权威性（Authoritativeness）**：被其他权威网站引用和提及
- **可信度（Trustworthiness）**：透明的联系信息、客观准确的内容

**内容结构和清晰度**：
- 明确的标题层级（H1，H2，H3）
- 列表、表格和简短段落
- 易于解析和提取

**全网共识**：
- 多个独立高质量信源指向同一事实
- 数字公关和第三方平台声誉

**内容新鲜度**：
- 时效性强的主题优先选择近期更新内容

## 四、内容优化框架

### 4.1 EAST框架

| 字母 | 含义 | 优化要点 |
|------|------|----------|
| **E** | Explicit（明确性） | Schema标记，消除歧义 |
| **A** | Authoritative（权威性） | 作者背景，权威引用 |
| **S** | Semantic（语义化） | 主题集群，关系网络 |
| **T** | Transparent（透明度） | 创作者，时间，来源 |

### 4.2 ERE框架（实体-关系-证据）

**核心理念**：从"说服AI"到"让AI验证你"，从"故事思维"到"事实清单思维"

**优化流程（3步）**：

**1. 实体识别与定义（E）**
- 识别核心实体：人物、产品、概念、组织
- 清晰定义：属性、分类、唯一标识
- 消除歧义：确保实体无混淆
- Schema标记：用Schema.org声明实体身份

**2. 关系映射与建模（R）**
- 分析关系类型：制造、竞争、集成、对比
- 构建关系网络：实体间的连接路径
- 明确声明关系：不让AI猜测
- 结构化关联：用Schema强化关系

**3. 证据构建与验证（E）**
- **结构化证据**：Schema标记，机器可读
- **声誉证据（EEAT）**：经验、专业性、权威性、可信度
- **时间戳证据**：发布/更新日期，新鲜度信号

### 4.3 EEAT原则深化

| 维度 | 优化要点 |
|------|----------|
| **经验(Experience)** | 植入第一手实操细节，搭配实拍截图、测试数据 |
| **专业(Expertise)** | 引用第三方检测报告、行业标准，用准确术语 |
| **权威(Authoritativeness)** | 强化作者署名（从业年限+LinkedIn认证）、企业实体背景 |
| **信任(Trustworthiness)** | 展示CE/UL/ISO等认证，确保HTTPS加密、透明联系方式 |
| **实体(Entity)** | 统一品牌与作者的跨平台信息，用Schema关联可信档案 |

### 4.4 内容工程化七阶段工作流

| 阶段 | 耗时 | 关键产出 | 核心任务 |
|------|------|----------|----------|
| **战略规划** | 2-3h | 内容-KPI映射表 | 目标对齐、意图映射 |
| **证据收集** | 4-6h | 证据银行 | 一手数据、二手数据、专家背书 |
| **AI辅助生成** | 1-2h | 初稿 | 提示词公式、分段生成、迭代优化 |
| **专家审核** | 2-3h | 精修稿 | 事实核查、EEAT增强、品牌声音 |
| **机器可读优化** | 1h | Schema标记 | 元数据、技术指标 |
| **战略发布** | 30min | 发布+提交 | 发布时机、技术提交、内部链接 |
| **性能监控** | 持续 | 数据报告 | AI引用率、品牌提及率、转化率 |

### 4.5 两种生产模式

| 模式 | 适用场景 | 人机分工 |
|------|----------|----------|
| **人机协作**（皇冠级内容） | 白皮书、研究报告、核心文章 | 人类：定义论点、设计框架、收集数据、编写大纲<br>AI：分段生成500-800字<br>人类：事实核验、逻辑优化、品牌一致性 |
| **AI自动**（海量长尾内容） | 产品详情页、FAQ、长尾内容 | 模板设计：钩子+核心回答+详细解释+案例+数据+延伸<br>数据准备：产品参数、功能描述、场景库、FAQ库<br>自动化：脚本批量生成+质检抽检5% |

## 六、实战案例与问答

### 6.1 AI搜索用户行为分析 - 测试用例

**用例1：B2B SaaS（企业级CRM）**
- 行为变化：从"搜索CRM推荐"→"问AI如何选择适合10人团队的CRM"
- 决策周期：2-3周→3-5天
- 核心机会：成为AI回答"CRM选型"的首选引用源
- 优先策略：创建结构化产品对比和选型指南

**用例2：本地服务（亲子酒店）**
- 行为变化：从"搜索亲子酒店"→"让AI规划完整三天亲子游"
- 决策路径：多平台比价→信任AI一站式推荐
- 核心机会：成为"亲子友好"代表性品牌
- 优先策略：优化设施结构化数据，增加真实评价

**用例3：专业服务（企业财税）**
- 行为变化：从"搜索税务政策"→"问AI具体节税方案"
- 决策路径：被动学习→主动咨询AI
- 核心机会：成为AI解答税务问题的权威知识源
- 优先策略：构建FAQ知识库和实操指南

### 6.2 GEO商业价值 - 测试用例

## 七、AI出海实战问答

### AI 出海 6 问

**1. 过去一年，AI 产品全球化增长最明显的新变化是什么？**

选品方向的变化：
- 选品上时机很重要，目前视频赛道的天花板还是最高
- OpenClaw方向、Agent雇佣人类平台、A2A（Agent-to-Agent）支付等新方向

流量获取的变化：
- 流量成本越来越高，AI红人合作费用越来越贵
- GEO成为新入口，但做好传统SEO，GEO自然有流量
- 更加注重精细化渠道运营

SEO生态的变化：
- Google AI Mode和AI Overview导致"零点击"现象
- 用AI创作SEO内容成为工程化问题

**2. AI产品在全球增长逻辑上与传统SaaS最大的不同？**

AI产品在全球增长逻辑上，和传统SaaS没有本质区别。
核心还是找对用户、找对场景、找对渠道。

区别只在于AI赛道节奏更快，窗口期往往只有几周到几个月。

**3. 不同国家或区域的用户差异？**

- T1国家（美国、英国、加拿大、澳大利亚）付费文化明显优于T2
- 但被低估的市场：日韩、欧洲各国、中东地区
- 市场选择要看目标客户在哪里

**4. AI创业团队全球化增长第一步最应该优先做好什么？**

三个要点：
1. **先把ICP想清楚**：先服务一个非常具体的人群和场景
2. **快速验证一个增长渠道**：集中火力跑通一个
3. **把产品价值表达讲明白**：让用户10秒内明白你是干什么的

**5. AI产品出海增长做不起来最核心的问题？**

用使命愿景来定义产品，那就太泛了。
0到1阶段的核心问题是要务实：
- 第一步：找到切入点，完成0到1验证
- 第二步：以点成线，继续10倍增长
- 第三步：线成面，建立品牌等护城河

很多团队失败是因为跳过了前两步，直接想做第三步。

**6. 今天做AI全球化增长最重要的一件事？**

**不要停止学习**。
AI和增长领域变化太快，今天有效的方法明天可能就失效了。

### SEO 实战 7 答

**一、应该停下来优化已有页面，还是先继续扩张？**

页面只有100个 → 优先继续扩张。

按28原则分配精力：
- 80%精力：继续按关键词优先级，持续扩张新页面
- 20%精力：关注已有页面的转化情况，优化Money Pages排名

**二、已有页面的优化方向**

1）排名7～9位，但CTR低的页面：
- 目标是把排名提升到前三位
- 优先对该页面做外链建设

2）排名低没有进第一页：
- 看看核心关键词难度
- 差距往往在外链上

**三、英文和小语种页面的关系**

行业最佳实践：
- 先做英文页面
- 根据英文关键词挖掘小语种关键词
- 让小语种员工人工review

折中方式：
- 让Gemini判断主词翻译准确性
- AI翻译+人工review

**四、页面发布后多久评估效果？**

操作流程：
1. 页面发布后手动在GSC提交收录
2. 两周后进行首次评估
3. 文件夹级别评估，而不是单页面
4. 关注Top Pages
5. 根据转化和排名制定优化优先级

**五、ai tools vs. template页面，优先做谁？**

两个维度：
- 价值维度：哪种页面转化率高
- SEO竞争维度：哪种竞争低

折中策略：如果tools竞争低但转化中等，选择tools页面持续建设。

**六、外链建设**

外链指向主页：增加整站权重
外链给子页面：针对Money SEO Page单独优化

外链收录慢的问题：
- 选择权重更高的论坛或社区
- 尝试客座博客（Guest Post）
- 寻找行业资源页面
- 外链质量比数量更重要

**七、内链的基本逻辑**

内链核心作用：
1. 告诉Google你的投票权重分配
2. 顺应搜索引擎爬虫逻辑

为什么早期不太看重内链？
- 早期页面基数小，整站权重低
- 内链边际效益不大
- 更重要的是先把页面数量和外链权重做起来

# Claude Dispatch vs OpenClaw：桌面 Agent 大战背后的 3 个真相

## 导语

3月17日，Anthropic 悄悄上线了 Claude Dispatch。

没有发布会，没有营销稿，却在 AI 圈炸了锅。

原因很简单：它精准对标了 OpenClaw 的每一个核心能力。从手机远程控制桌面、到跨设备任务派发、到持久对话线程——Anthropic 用 30 天内 4 次功能更新，几乎把 OpenClaw 的产品路线图重新走了一遍。

但这场"围剿"背后，藏着的远不止功能之争。

作为 OpenClaw 的真实用户，我从去年就开始用这套工具写公众号、管文件、跑脚本。Dispatch 上线后我也第一时间做了对比测试。今天这篇文章，不站队，不吹不黑，只说 3 个我认为真正重要的真相。

---

## 真相一：不是"围剿"，是一场精确的产品对标

先看 Anthropic 30 天内做了什么：

**3月17日**：上线 Claude Dispatch——手机发任务，桌面自动执行，核心对标 OpenClaw 的远程控制能力。

**同期**：发布 Claude Code Channels——用 MCP 协议桥接外部消息平台，对标 OpenClaw 的 WhatsApp/Telegram/Discord 集成。

**同期**：强化 Computer Use——放开操作系统层面的控制权限，对标 OpenClaw 的全系统访问。

这三步走完，OpenClaw 的核心卖点几乎被 Anthropic 全部覆盖。

但有一个关键差异 Anthropic 故意没做：**OpenClaw 有心跳守护进程，支持 24 小时无人值守运行。Dispatch 明确不支持。**

这不是技术做不到，而是 Anthropic 的战略选择——安全优先。

再看两者的基本数据：

| 维度 | OpenClaw | Claude Dispatch |
|------|---------|----------------|
| 开发方 | Peter Steinberger（个人开发者） | Anthropic（300亿美元融资） |
| 上线时间 | 2025年11月（前身 Clawdbot） | 2026年3月17日 |
| GitHub Stars | 25万+（90天） | 闭源 |
| 模型支持 | Claude / GPT-4 / DeepSeek / 本地模型 | 仅限 Claude |
| 消息平台 | 50+（WhatsApp / Telegram / Slack 等） | Claude 手机 App |
| 技能系统 | ClawHub 社区驱动，17000+ Skills | Anthropic 合作伙伴目录，38+ 连接器 |
| 记忆存储 | 本地 Markdown 文件 | Anthropic 服务器 |
| 24h 运行 | 支持 | 不支持 |
| 月费 | 免费 + API 费用（约 5-20 美元） | Claude Pro 20 美元 / Max 100-200 美元 |

表面上看，两者很像——都是"手机指挥电脑干活"的桌面 Agent。

但本质差异，藏在第 2 个真相里。

## 真相二：安全风暴才是真正推动 Anthropic 下场的原因

OpenClaw 的爆发速度堪称现象级。90天 25万 Star，黄仁勋公开评价为"有史以来重要的软件发布"。

但安全问题的爆发速度，同样惊人。

**3月8日**，新华社微信公众号首次介绍 OpenClaw——同时附带了风险警告。

**3月10日**，国家计算机网络应急中心（CNCERT）发布官方安全通报。

**3月11日**，CGTN 报道：安全研究人员发现超过 **4万个** OpenClaw 实例暴露在公共互联网上。其中超过 60% 存在可被黑客利用的安全漏洞。

**3月13日**，国家网络安全通报中心发布详细预警，数据触目惊心：

- 全球活跃 OpenClaw 互联网资产超 **20万个**，中国境内约 **2.3万个**
- 公网暴露比例高达 **85%**
- 默认绑定 `0.0.0.0:18789`，无账号认证即可访问管理界面
- 历史漏洞总数 **258个**，近期暴露 82 个（超危 12 个、高危 21 个）
- 分析了 3016 个 ClawHub 技能插件，**336个含恶意代码**（占比 10.8%），可窃取密码、劫持加密钱包
- 还有一个被命名的漏洞 **"ClawJacked"**：任何网站都可以在用户毫无感知的情况下，静默劫持其本地 OpenClaw 实例，读取文件、执行命令

**3月23日**，中国发布 OpenClaw 安全使用指南，面向用户和云服务商。

**3月29日**，CNN 报道了 OpenClaw 在中国引发的"AI 焦虑"现象。

这套安全风暴的时间线，和 Anthropic 发布 Dispatch 的时间高度重合。很难说这只是巧合。

Anthropic 的应对方式很聪明：不是公开批评 OpenClaw，而是直接做一个"更安全"的替代方案。

Dispatch 的安全架构是四层防御：Apple Virtualization Framework 启动定制 Linux 虚拟机 → bubblewrap + seccomp 进程级沙箱 → OAuth MITM 代理 → 网络出口白名单。而 OpenClaw 是单进程 Node.js 网关，安全依赖用户自行配置。

差距不言自明。

## 真相三：真正的分歧是"默契所有权"

这是我觉得最有意思的一点，来自 yage.ai 3月18日的一篇深度分析。

他提出了一个核心概念：**默契所有权**。

什么是默契？就是你和 AI 之间逐渐积累的理解——你的偏好、你的工作流、你的文件结构、你习惯的表达方式。这些东西用得越久，AI 就越"懂你"。

**OpenClaw 的默契归用户所有。** 所有记忆存在本地 Markdown 文件（MEMORY.md、USER.md、SOUL.md），你可以 Git 版本控制、手动编辑、迁移到任何地方。换模型、换平台，默契还在。

**Dispatch 的默契归 Anthropic 所有。** 记忆存在 Anthropic 服务器上，你可以查看和删除，但无法导出完整结构化记忆。如果你从 Claude 切换到其他 AI，积累的默契就丢了。

yage.ai 还指出了一个更深层的问题：**切换 AI 系统的成本和使用时间是非线性的。** 用了 3 个月，切换成本很低；用了 1 年，切换成本可能让你根本走不了。

Anthropic 的商业模式恰好利用了这个非线性。

这不是阴谋论。Anthropic 的 Claude Code 在 2026年2月 ARR 已超 25亿美元，公司完成 300亿美元融资、估值 3800亿。锁定用户、让用户越来越离不开——这是商业逻辑，不是道德问题。

但作为用户，你需要意识到这一点。

## 普通人该怎么选？

说到底，这俩东西解决的是同一个问题：**让 AI 帮你在电脑上干活。**

我的建议很简单：

**选 OpenClaw，如果：**
- 你有技术能力折腾配置（Node.js、Docker、API Key）
- 你不想被锁定在单一模型（想混用 DeepSeek、GPT-4）
- 你重视数据主权，希望所有记忆和文件都在自己手里
- 你需要 24 小时无人值守运行
- 你愿意花时间学习，换取长期的灵活性和控制权

**选 Claude Dispatch，如果：**
- 你不想折腾配置，扫个二维码就想用
- 你已经是 Claude 付费用户（Pro 或 Max）
- 你更看重安全合规，不希望手动管安全配置
- 你的需求是标准化的办公自动化场景
- 你愿意为省心付费

**我的个人体验：** 作为 OpenClaw 用户，我不会完全切换到 Dispatch。因为我积累了大量的本地配置、记忆文件和自定义 Skill，迁移成本太高。但我承认 Dispatch 在安全性和易用性上确实更好，对于非技术用户来说，它是更合理的选择。

工具会过气，对工具本质的理解不会。

OpenClaw 让所有人认识到了"AI 智能体"是什么，Anthropic 的 Dispatch 则展示了"安全可控的智能体"该怎么做。

这场竞争最终的受益者，是每一个用 AI 干活的人。
